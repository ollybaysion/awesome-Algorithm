# 알고리즘 학습 개요

## 왜 알고리즘을 공부하는가?

알고리즘은 **문제를 효율적으로 해결하는 단계적 절차**입니다. 좋은 알고리즘은:

- 시간 복잡도를 줄여 **빠른 실행** 속도를 보장합니다.
- 공간 복잡도를 최적화해 **메모리 효율**을 높입니다.
- 코딩 테스트, 기술 면접, 실무 문제 해결의 **핵심 역량**입니다.

---

## 학습 로드맵

```
자료구조 → 정렬 → 탐색 → DP → 그리디 → 그래프 → 문자열
```

| 단계 | 주제 | 난이도 |
|------|------|--------|
| 1 | 자료구조 기초 | ⭐ |
| 2 | 정렬 알고리즘 | ⭐⭐ |
| 3 | 탐색 알고리즘 | ⭐⭐ |
| 4 | 동적 프로그래밍 | ⭐⭐⭐ |
| 5 | 그리디 알고리즘 | ⭐⭐⭐ |
| 6 | 그래프 알고리즘 | ⭐⭐⭐⭐ |
| 7 | 문자열 알고리즘 | ⭐⭐⭐ |

---

## 시간 복잡도 빠른 참조

| 복잡도 | 표기법 | 예시 |
|--------|--------|------|
| 상수 | O(1) | 배열 인덱스 접근 |
| 로그 | O(log n) | 이진 탐색 |
| 선형 | O(n) | 선형 탐색 |
| 선형로그 | O(n log n) | 합병 정렬, 퀵 정렬 |
| 이차 | O(n²) | 버블 정렬, 선택 정렬 |
| 지수 | O(2ⁿ) | 완전 탐색 (부분집합) |
